## Metodología SDD

Este proyecto sigue la metodología **Specs Driven Development (SDD)**, que prioriza la creación de especificaciones detalladas antes de la implementación.

### Principios de SDD Aplicados

1. **Specs First**: Toda funcionalidad debe estar documentada antes de codificarse
2. **Revisión de Pares**: Las specs son revisadas por el equipo antes de implementación
3. **Versionamiento**: Las specs se versionan junto con el código
4. **Traza bi-directional**: Cada requisito se puede trazar a código y viceversa

## Tecnologías del Proyecto

- **Frontend**: Flutter 3.4 (Material Design)
- **Backend**: Node.js v20, Express 5.0
- **Base de Datos**: MySQL 8.0
- **Infraestructura**: VPS con HTTPS/SSL
